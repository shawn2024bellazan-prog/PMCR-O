# Architecture Decision Records

*Author: Shawn Bellazan · ThoughtLock: 2026-05-05*

Architecture Decision Records document why the substrate is structured the way it is. Each ADR captures the context that forced a decision, the decision itself, and the consequences — positive, negative, and neutral.

ADRs are immutable once accepted. A superseded ADR is not deleted — it is marked Superseded and the new ADR references it.
